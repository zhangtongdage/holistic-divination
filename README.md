## 项目背景

中华传统术数（六爻、梅花易数、奇门遁甲、子平八字等）各有体系，彼此之间存在壁垒。本项目首次提出**「全息人本」**理念——一切卜算的本质是**人、时、空、事、念**五维信息的共振映射，而非单一术数体系的机械套用。

### 核心创新

| 创新点 | 说明 |
|--------|------|
| 六层融合架构 | 公理→体用→机变→取象→义理→占断，统一多术数体系 |
| 七维度密钥 | 本命四柱+时空干支+汉字笔画+境地五行+心念权重+经纬度+行为特征 |
| 本地AI推理 | Qwen2-1.5B QLoRA微调，100%离线运行 |
| 典籍溯源 | 卦辞/彖传/象传/爻辞自动匹配，有据可查 |
| 唯一性保证 | 任何变量变化都导致完全不同结果，真正千人千面 |

## 快速开始

### 环境要求

- Node.js 18+
- npm 或 pnpm

### 1. 克隆项目

```bash
git clone https://github.com/zhangtongdage/holistic-divination.git
cd holistic-divination
```

### 2. 安装依赖

```bash
# 核心引擎
cd core && npm install

# 桌面端(可选)
cd ../desktop && npm install

# 移动端(可选)
cd ../mobile && npm install
```

### 3. 下载AI模型

模型文件约950MB，需手动下载：

```bash
mkdir -p core/models

# 从 GitHub Release 下载
# https://github.com/zhangtongdage/holistic-divination/releases
# 将 xuanji-1.5b.gguf 放入 core/models/
```

### 4. 构建运行

```bash
cd core
npm run build    # 构建核心引擎
npm run dev      # 开发模式
```

## 推理模式

### 本地模式（推荐）
- ✅ 完全离线，无需网络
- ✅ 隐私安全，数据不出本地
- ✅ 基于Qwen2-1.5B微调模型

### API模式
- ✅ 支持OpenAI/Claude/智谱/DeepSeek
- ✅ 解读质量更高
- ⚠️ 需要网络和API密钥

### 混合模式（默认）
- 优先使用本地模型
- 本地失败自动回退API
- 兼顾速度和质量

## FAQ

**Q: 模型文件太大，如何减小？**
A: 可以使用更激进的量化(如Q3_K_S)，但会降低解读质量。建议保持Q4_K_M。

**Q: 没有GPU能跑吗？**
A: 可以，CPU推理也能用，只是速度较慢(约5 tokens/s)。

**Q: 支持哪些术数体系？**
A: 六爻纳甲、梅花易数、奇门遁甲、子平八字，后续会增加紫微斗数、铁板神数等。

**Q: 训练数据从哪来？**
A: 基于《增删卜易》《卜筮正宗》《四库全书·术数类》等古籍生成的问答对。
