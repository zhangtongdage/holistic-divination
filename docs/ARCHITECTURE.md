# 架构设计文档

## 概述

全息人本古法卜算系统采用三层架构设计，实现从核心算法到用户界面的完整闭环。

## 系统架构

### 1. 核心引擎层 (core/)

核心引擎是整个系统的中枢，采用纯TypeScript实现，零外部依赖。

#### 1.1 全息信息采集模块

**职责**：收集用户的多维度信息，构建全息画像。

```
信息层次：
├── 必填层（3层）
│   ├── 基础人象：生辰八字、性别、出生地
│   ├── 核心问事：问题类别、具体描述、紧急程度
│   └── 当前境地：最近经历、环境因素
│
└── 可选层（3层）
    ├── 发问状态：心态、期望、疑虑
    ├── 补充人象：职业、家庭、健康
    └── 主观预期：倾向性、偏好
```

#### 1.2 唯一性密钥生成层

**职责**：基于七维度信息生成唯一的卜算密钥。

```typescript
// 七维度混合同余算法
interface DivinationKey {
  bazi: string;        // 本命四柱（年月日时）
  timeGanZhi: string;  // 时空干支（起卦时间）
  strokeHash: number;  // 汉字笔画哈希（问题关键词）
  wuxing: string;      // 境地五行（环境因素）
  mindWeight: number;  // 心念权重（心理状态）
  geoHash: number;     // 经纬度哈希（地理位置）
  behavior: number;    // 行为特征（历史交互）
}
```

**唯一性保证**：
- 任何维度的微小变化都会导致完全不同的密钥
- 采用混合同余算法确保均匀分布
- 支持2^64种可能的密钥组合

#### 1.4 混合AI推理层

**职责**：提供智能解读能力，支持本地和云端双模式。

```typescript

type AIMode = 'local' | 'api' | 'hybrid';

// 本地推理（推荐）
```

### 2. 应用层

#### 2.1 桌面端 (desktop/)

- **技术栈**：Tauri 2.0 + Vue 3 + Pinia
- **平台**：Windows / macOS / Linux
- **特性**：
  - 原生性能，启动速度快
  - 本地SQLite存储
  - 系统托盘支持
  - 自动更新

#### 2.2 移动端 (mobile/)

- **技术栈**：UniApp X
- **平台**：Android / iOS / 小程序
- **特性**：
  - 一套代码多端运行
  - 原生渲染性能
  - 离线缓存支持
  - 推送通知

### 3. 数据层

#### 3.1 存储设计

```typescript

interface SecureStorage {
```

#### 3.2 隐私保护

- 所有数据本地存储，零云端上传
- 敏感信息AES-256加密
- 支持数据导出和清除
- 符合GDPR/个人信息保护法

## 安全设计

1. **数据加密**：AES-256-GCM加密所有敏感数据
2. **密钥管理**：密钥派生自设备特征，不存储明文
3. **网络安全**：API通信使用TLS 1.3
4. **代码安全**：依赖项定期审计，无已知漏洞

## 扩展性设计

1. **术数体系插件化**：新体系可作为独立模块接入
2. **AI模型可替换**：支持更换不同的基座模型
3. **多语言支持**：国际化框架已预留
4. **云端同步**：可选的端到端加密同步
